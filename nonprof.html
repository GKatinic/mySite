Empathize Phase

We began working on this non-profit organization project by conducting user research. First the team & I have brainstormed on some survey questions that we sent out 25 of over social media platforms. While waiting on the the surveys to return we conducting a UI Accessibility Test & Heuristics Evaluation, were we discovered that the current site has many UI related problems. Some of the issues we found are that the website has no breadcrumbs, outdated iconography, inconsistent way home, hidden search box & cart, no error messages for issues & one error message found with an incorrect message, correctly filled out boxes keep red stroke which makes it look incorrectly filled like the example in the picture below.

Next we completed a Direct & Indirect Competitor Analysis to collect data on their competitors. The analysis has come up with some great insights such as:

  • Clear communication on how organization uses donations.

  • Multiple payment Options (Credit, Debit, PayPal, etc.)

  • Secure Payment Processing.

  • Easy to Locate Sites Main Objectives.


After receiving the 25 Surveys back, we started analyzing the data & turned it into a Data Sheet that we turned into the 2 graphs that are showing below. The graph on the left shows us that 65.2% donations payment are made with Credit or Debit Cards & only 34.8% are using Venmo, PayPal, Cash, Etc.

Next we completed multiple User Test on the Original Primary Navigation, Donations, Volunteering, Shopping, & Checkout. Conducting the tests we discovered that users are haing a hard time locating their main objective & that they felt really nervous giving out personal information.


  • Hard to locate main objectives.

  • Site has untrustworthy look & feel.

  • Not a single tester was comfortable giving out their payment information.

  • Misunderstanding UI features.

Define Phase

After conducting The User Research we now moved to the define phase of of our process. First we collected the User Insights & then created an Affinity Diagram. The diagram really showed us that users really want to know where their donations are going & if a website that they are donating to does not feel safe they will not donate. The website needs to have more payment options such as single or reoccurring. Once we finished with the diagram we were able to create a User Person.

Our User Persona focuses on the generous donator & volunteer who will be visiting the website to do any of the 3 main objectives of the site, which are donating, shopping, & volunteering.
Janice is a 62 year old widow who loves to spend most of her free time since her husband passed away helping local native tribes by volunteering her time, donating her hard earned money, or shopping for native hand craft items to resell at her souvenir shop.

Next we did a User Journey Map which shows an example of a user on the website indecisive to donate because of the many UI issues she is experiencing.

Ideation Phase

We brainstormed for ideas using the “I Like, I Wish, What If” exercise. This led us to notice that there are only a few thinks we actually liked about the website & that there would be a lot of re-design needed. So we had to come up with some great ideas to help us out. Then we prioritized our ideas by complexity & impact with the Prioritization Matrix Chart. So now that we see on the chart what’s more impactful with the least amount of complexity we can prioritize that by the time available for the project.

Some Ideas:

  • More Ways to Donate.
  • URL Matched the Cause.
  • More Volunteer Hour Options.
  • More High Def Imagery Used Correctly.
  • Creating an Account Option.
  • Check Box for Billing & Shipping Pairing.

Next we have created a User Flow, to show the path the user takes navigating through the website to complete a donation.

We took the original sites navigation & re-organized & re-labeled some content through The Card Sorting exercise. Which pleasing to say looks a lot sharper with a lot clear labeling.

  • Adopt an Elder >> Support an Elder
  • Information on Items and Pricing >> Products & Pricing
  • Volunteer Opportunities >> Volunteer

After we conducted the card sorting on the navigation we framed it into a organized site map structure with 4 primary navigation with dropdowns & a Donate Button on the right hand side of the header.

Next we sketched out a few proto-types of the website the new navigation. Then we took our favorite parts of the sketches turn them into a new sketch of the desktop, mobile, tablet. Next we re-did our user tests from the original site to verify our work which came out to be a great success. Now we took the paper sketches and turned them into low fidelity proto-types using Adobe XD. Then to mid fidelity which you see right below this text, then finally polished it off with the high fidelity responsive web design proto-type you’ll see below in the photo gallery.

It all began with a Non-Profit Organization website called Adopt-A-Native-Elder.

The website had many issues which made it very pleasant to work on. It really feels good to lend a helping hand to an organization that gives it its all to help others in need. I hope my re-design will help increase the sites volunteers, shoppers, donations. I have learned a lot through this project about myself, the importance of UI Design, especially the user his want needs.
